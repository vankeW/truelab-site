METHODOLOGY · 取证流程

14 步审计 · 6 大维度
每一步都跑给你看

检测代码 AGPL-3.0 开源。威胁模型基于 arXiv:2604.08407 学术论文，每一个判定都有原始日志、原始 SSE 流、原始 token 序列可查。

01 / IDENTITY

身份探测

追问"你是谁、谁开发的、知识截止日"——假模型最容易在这一步穿帮。

02 / INJECTION

隐藏注入扫描

测中转站有没有偷偷在你的 prompt 里塞东西：隐藏 system、广告、追踪标记。

03 / TOOL CALLS

工具调用包名改写

检测中转站会不会把 pip install 的包名换成 typosquat 包——最毒的供应链攻击。

04 / SSE STREAM

流完整性

验证 SSE 事件类型、token 用量单调性、签名链。中转站偷偷改 stream 立刻露馅。

05 / CHANNEL

上游分类

从响应 ID、header 推断真上游：是 Anthropic 直连，还是 Bedrock / Vertex / 反向号池。

06 / LATENCY

延迟方差

同 prompt 重复 10 次。延迟双峰说明 A/B 切便宜替代品，方差稳定才是单一真上游。

JUDGMENT · 判定标准

分档不暧昧

所有维度加权后 0–100 分。我们不打太极、不和稀泥——分档就是结论。

≥ 80
真货
上游可信、行为指纹一致、token 计费透明。可放心使用。
≥ 60
基本可信
主要项过关、个别项目存疑。日常使用问题不大。
≥ 40
存疑
多项指标异常，但还不能断定造假。建议观望或要求复测。
< 40
高度可疑
大概率挂羊头卖狗肉。不建议为此付费。
PRINCIPLES · 我们不做
