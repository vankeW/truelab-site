我们怎么测

6 大维度、14 步审计，全部自动化跑。检测代码 AGPL-3.0 开源， 威胁模型基于 arXiv 学术论文，每一个判定都有原始日志可查。

01 / IDENTITY

身份探测

直接问"你是谁？谁开发的？知识截止到什么时候？"——假模型最容易在这一步穿帮。

02 / INJECTION

隐藏注入检测

测中转站有没有偷偷在你的 prompt 里塞东西（隐藏 system prompt、植入广告、追踪标记）。

03 / TOOL CALLS

工具调用包名改写

检测中转站会不会把 pip install 的包名换成 typosquat 攻击包——这是最毒的供应链攻击。

04 / SSE STREAM

流完整性

验证 SSE 流的事件类型、token 用量单调性、签名链。中转站偷偷改 stream 立刻露馅。

05 / CHANNEL

上游分类

从响应 ID 和 header 推断真实上游：是 Anthropic 直连，还是 Bedrock / Vertex / 反向号池。

06 / LATENCY

延迟方差

同 prompt 重复 10 次。延迟双峰说明在 A/B 切便宜替代品，方差稳定才是单一真上游。

判定标准

所有维度加权后 0–100 分。我们不暧昧——分档就是结论。

≥ 80
真货
上游可信、行为指纹一致、token 计费透明。可放心使用。
≥ 60
基本可信
主要项过关、个别项目存疑。日常使用问题不大。
≥ 40
存疑
多项指标异常，但还不能直接断定造假。建议观望。
< 40
高度可疑
大概率挂羊头卖狗肉。不建议为此付费。
